CNN and its Architectures

  1. Introduction

    Why CNN came into existence

    CNNs were introduced out of necessity. Traditional neural networks (ANNs) struggled when faced with image data. They treated images as a flat list of numbers, which made it hard to identify patterns, textures, and spatial relationships.

    Let’s consider a simple example:

    A colored image of size 100×100 pixels has 3 color channels (RGB). That’s 100 × 100 × 3 = 30,000 input values. If we connect this to 100 neurons in the first hidden layer of a traditional ANN, we already have 30,000 × 100 = 3 million trainable parameters and that’s just for a single layer.

    Such huge parameter counts lead to:

    • Increased computational cost
    • Overfitting
    • Loss of important spatial features

    CNNs addressed these challenges by introducing convolutions, which drastically reduced parameters while preserving spatial structure.

  2. Prerequisites

    1. Artificial Neural Networks (ANNs)

      Artificial Neural Networks (ANNs) are computer models inspired by how the human brain works. They are made up of layers of connected units called neurons. Each connection has a weight that controls how strong the signal is. Data moves through these layers, and each neuron does a simple calculation before passing the result to the next layer.

      During training, the network learns by adjusting these weights to make better predictions. ANNs are the basic building blocks of deep learning and help us understand complex data patterns.

      Key Components of ANN

      Input Layer: This is where the network receives information. For example, in an image recognition task, the input could be an image.

      Hidden Layers: These layers process the data received from the input layer. The more hidden layers there are, the more complex patterns the network can learn and understand. Each hidden layer transforms the data into more abstract information.

      Output Layer: This is where the final decision or prediction is made. For example, after processing an image, the output layer might decide whether it’s a cat or a dog.

      Source: GeeksforGeeks

    2. Padding

      When an image goes through convolution (a process that looks for patterns), its size usually gets smaller. This shrinking can cause important details to be lost.

      To avoid this, we add extra pixels around the edges of the image — this is called padding. For example, if padding = 1, it means adding one row of pixels at the top and bottom, and one column on each side. This helps keep the image size bigger after processing.

      Source: GeeksforGeeks

    3. Striding

      Stride is how much the filter (a small window that scans the image) moves or "jumps" over the image when processing it. For example, if the stride is 1, the filter moves 1 pixel at a time and calculates the output at each position. If the stride is larger, like 2, the filter moves 2 pixels at a time, which makes the output smaller.

    4. Pooling

      Pooling is a way of reducing (downsampling) the features of an image while keeping the important information. This helps make the network faster and prevents overfitting.

      Types: Min, Max, Average, L2, and Global Pooling.

      Source: ResearchGate

  3. What is a Convolutional Neural Network (CNN)?

    A Convolutional Neural Network (CNN) is a special type of neural network mainly used for working with images. Unlike a traditional neural network that looks at all pixels at once, a CNN looks at small parts of the image at a time and detects patterns like edges, colors, and shapes.

    These smaller patterns are then combined step-by-step to form bigger and more complex features — such as objects or faces in a photo. This makes CNNs especially powerful for image classification, object detection, and many other computer vision tasks.

    Think of it like how humans look at a picture: first noticing simple shapes and colors, then putting them together to recognize what’s in the scene.

  4. Building Blocks of CNN

    1. Convolutional Layer:

      It acts as a pattern detector. Instead of looking at the whole image at once, it uses small grids called filters (or kernels) that scan across the image, piece by piece.

      • Each filter is designed to find a specific kind of feature — for example, vertical edges, horizontal edges, curves, or textures.
      • As the filter slides over the image, it performs a convolutional operation (dot product) between its weights and the pixel values in the current patch.
      • The result after the dot product is called a feature map, which highlights where that specific pattern appears in the image.

      Over multiple layers, these filters start detecting more complex things:

      • First layer → edges and simple shapes.
      • Middle layers → patterns like eyes, wheels, textures.
      • Deeper layers → entire objects like faces or cars.

      Feature Map Dimension Formula:

      The output size (dimension) of a feature map can be calculated as:

      (n + 2p - f) / s + 1
      • n → Size (width/height) of the input image.
      • p → Padding (extra pixels added around the image).
      • f → Filter (kernel) size.
      • s → Stride (number of pixels the filter moves at each step).

      Source: Anh Reynolds

    2. Activation Functions:

      An activation function is a small mathematical formula that decides whether a neuron should be “active” or not, and how much signal it should pass forward in a neural network.

      How it Works:

      • Takes the inputs → The neuron receives signals from previous neurons (e.g., x1, x2, x3).
      • Multiply by weights → Each input is multiplied by its connection weight (e.g., w1, w2, w3).
      • Add them up → The weighted inputs are summed: 
        z = (x1 × w1) + (x2 × w2) + (x3 × w3)
      • Add the bias → A bias term is added to shift the sum: 
        z = z + b
      • Pass through activation function → The activation function checks how strong the signal is.
      • Inactive example (ReLU) → 
        ReLU(z) = max(0, z)
        If z = -2.3 → output is 0. If z = 5.1 → output is 5.1.
      • Send to next layer → The output from the activation function becomes input to neurons in the next layer.

      The main reason we use activation functions is to add non-linearity. Without them, the network would only learn simple straight-line relationships between inputs and outputs.

      Some commonly used activation functions include:

      • Sigmoid — outputs values between 0 and 1.
      • Tanh — outputs values between -1 and 1.
      • ReLU (Rectified Linear Unit) — keeps positive values, turns negative ones to zero.
      • Leaky ReLU — similar to ReLU but allows small negative values.
      • Softmax — often used in the output layer for classification problems.

    3. Pooling Layer:

      We already gave a short intro to pooling in the prerequisites section. Let’s now understand it in detail.

      Pooling is a way of shrinking the feature map while keeping the most important parts.

      Why pooling is used:

      • Speeds up computation: Less data for the network to process.
      • Reduces overfitting: Makes the network less likely to memorize training data.
      • Gives spatial invariance: Features are recognized even if they move slightly.

      How it works:

      • A pooling layer slides a small window (e.g., 2×2) over the feature map
      • Instead of multiplying and summing like in convolution, it performs a simple calculation in each window.
      • The output is a smaller version of the feature map, but with important patterns preserved.

      Example (Max Pooling, 2×2 kernel, stride = 2):

      [ 1   3   2   4 ]
[ 5   6   7   8 ]
[ 9   2   0   1 ]
[ 4   5   6   7 ]
      • First 2×2 block → max is 6
      • Second 2×2 block → max is 8
      • Third 2×2 block → max is 9
      • Fourth 2×2 block → max is 7
      [ 6   8 ]
[ 9   7 ]

      Common types of pooling:

         
      1. Max Pooling: Keeps the largest value in each window.
        2.  
      2. Average Pooling: Takes the average in each window.
        3.  
      3. Global Pooling: Reduces the entire feature map to a single value.

    4. Fully Connected Layer

      A fully connected layer is a traditional type of neural network layer where every neuron is connected to every neuron in the previous layer. Its main job in a CNN is to take the high-level features extracted by the convolutional and pooling layers and use them to make a final prediction.

      After the convolutional and pooling layers have done their job of finding and simplifying all the important features in an image (like edges, shapes, or textures), this layer takes all that information and uses it to make a final decision. Think of it as the final stage of the network, where all the learned features are combined to make a confident classification.

      How It Calculates Its Output

      The fully connected layer operates in a few key steps:

      • Flattening: Before the data can enter the fully connected layer, the 2D or 3D feature maps from the last pooling layer are flattened into a single, long line of numbers (a vector). This process essentially converts the spatial data into a one-dimensional format suitable for a traditional neural network.
      • Connecting: This flattened vector is then fed into the fully connected layer. Every single value in the vector is connected to every neuron in this new layer.
      • Making a Prediction: Each neuron in the fully connected layer performs a weighted sum of all the inputs it receives. The output of a single neuron is calculated using the following formula: 
       y = f( Σi=1n wixi + b )

      Source: GeeksforGeeks

      • y is the output of the neuron.
      • f is the activation function (e.g., ReLU or Softmax).
      • wi is the weight of the connection from the i-th input.
      • xi is the value of the i-th input from the flattened vector.
      • b is the bias term, which shifts the activation.
      • The summation (Σ) is performed over all n inputs from the previous layer.

      The final layer often uses a function like Softmax to turn these outputs into probabilities for each possible class. For example, in an image classification task, the output might show an 80% probability for "dog," 10% for "cat," and 10% for "bird."

      In short, the convolutional and pooling layers find the puzzle pieces, and the fully connected layer puts them all together to see the final picture and make a decision about what it is.
